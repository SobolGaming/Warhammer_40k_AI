# uv.lock generation

A real `uv.lock` file must be generated by `uv`; it should not be hand-written.

The sandbox used to prepare this starter bundle could not create the lockfile because it had no network/DNS access and `uv` attempted to download CPython 3.14 from GitHub.

Generate it in the new repository with:

```bash
uv python install 3.14.5
uv lock
uv sync
uv lock --check
```

Then commit the resulting `uv.lock`.

Why this is not included as a fake file:

- `uv.lock` contains exact package resolutions, artifacts, hashes, markers, and dependency metadata.
- A fabricated lockfile would be worse than no lockfile because it could appear reproducible while being invalid.
- `uv.lock` is managed by uv and should not be edited manually.
